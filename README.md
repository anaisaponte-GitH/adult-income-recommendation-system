# Adult Income Recommendation System

## Overview

This project builds a content-based recommendation system using the Adult Census Income dataset.

Instead of simply predicting whether an individual earns more than $50,000 per year, the solution identifies people with similar profiles who belong to a higher-income group and generates interpretable recommendations based on those similarities.

The project demonstrates how Machine Learning can move beyond prediction and support decision-making through personalized recommendations.

---

## Business Problem

Organizations often need to understand which characteristics are associated with higher income levels in order to:

- Identify socioeconomic patterns.
- Support workforce and education analysis.
- Generate personalized recommendations.
- Improve decision-making through data-driven insights.

Rather than focusing only on classification, this project explores how recommendation techniques can help individuals understand potential paths associated with higher-income profiles.

---

## Dataset

The project uses the Adult Census Income Dataset, which contains demographic, educational and employment-related information.

### Features

Examples of variables include:

- Age
- Education level
- Occupation
- Hours worked per week
- Marital status
- Relationship
- Capital gain
- Capital loss
- Workclass

### Target Variable

Income category:

- Income > $50K
- Income ≤ $50K

---

## Methodology

### 1. Data Preparation

Data preprocessing included:

- Data cleaning.
- Handling missing values.
- Replacing unknown categories.
- Feature encoding using One-Hot Encoding.
- Feature scaling using StandardScaler.
- Removal of redundant variables.

### 2. User Profile Definition

Each individual in the dataset was treated as a user profile.

Profiles were represented using demographic, educational and employment-related attributes.

The dataset was divided into two groups:

- Individuals earning more than $50K.
- Individuals earning $50K or less.

### 3. Content-Based Recommendation System

A recommendation engine was built using profile similarity.

The approach consisted of:

1. Representing each user as a feature vector.
2. Calculating similarity between profiles.
3. Comparing lower-income individuals against higher-income profiles.
4. Generating recommendations based on observed differences.

### 4. Similarity Calculation

Cosine Similarity was used to identify the most similar higher-income individuals for each target user.

This allows the system to find profiles with comparable characteristics and extract meaningful patterns associated with higher income levels.

### 5. Recommendation Generation

Recommendations were generated by comparing:

- Education level
- Occupation
- Working hours
- Employment characteristics

The system highlights attributes that frequently appear among similar higher-income individuals.

---

## Example Recommendation Logic

For a user earning less than $50K, the system:

1. Finds the most similar profiles earning more than $50K.
2. Identifies relevant differences between profiles.
3. Generates interpretable recommendations based on those differences.

Examples may include:

- Higher education levels.
- Different occupational categories.
- Greater weekly working hours.
- Alternative employment characteristics.
